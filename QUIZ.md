# LearnLoop — Quiz & Coding Practice System

**Version:** 1.0  
**Depends on:** Phase 5 complete (AI Q&A must be working)  
**Build after:** Phase 5 — Add as Phase 5.5 before Phase 6  
**AI Provider:** Google Gemini API (gemini-1.5-flash — free tier)  
**Goal:** Prevent tutorial hell — user actually retains what they learned

---

## Why This Feature Exists

Tutorial hell = user watches 10 hours of content, feels productive, but can't build anything.  
This quiz system forces **active recall** after every video — the most effective learning technique.  
It's optional — user can skip — but completing quizzes earns bonus progress on their profile.

---

## How It Works (User Flow)

```
User watches video → 90% complete → video marked done
        ↓
Popup appears (bottom of screen, non-blocking):
"Test yourself on this video? Takes 2 min  [Start Quiz]  [Skip]"
        ↓
If user clicks Skip → nothing happens, video stays complete
        ↓
If user clicks Start Quiz:
  → Gemini API generates questions from video transcript
  → Quiz modal opens with questions one by one
  → After last question → results shown
  → Quiz score saved to DB
        ↓
When ALL videos in a playlist are complete:
  → Final Course Quiz triggers (same popup style)
  → This quiz covers the entire course (harder questions)
  → Includes coding practice questions
  → Completing final quiz → unlocks certificate
  → (If user skipped all per-video quizzes, certificate still unlocks
     but final quiz is strongly encouraged)
```

## Question Types

### Per-Video Quiz (after each video)
- **3–5 questions per video** (not too long — user just finished watching)
- Question types:
  1. **MCQ** — 4 options, 1 correct
  2. **True/False** — simple concept check
  3. **Fill in the blank** — complete the statement
- All generated by Gemini from video transcript
- No coding questions here (keep it light)

### Final Course Quiz (after all videos complete)
- **8–12 questions total**
- Question types:
  1. **MCQ** — concept questions (3–4 questions)
  2. **Short explanation** — "Explain X in your own words" (1–2 questions)
     User types answer → Gemini evaluates it → gives score + feedback
  3. **Coding practice questions** — (3–5 questions)
     - Question shown with context
     - User clicks "Practice on..." → opens external platform
     - User marks it done manually after completing

### Coding Practice Questions — External Platform Links
```
Question shown:
"Write a function that reverses a string without using built-in methods."

Below the question:
[Practice on LeetCode →]  [Practice on CodePen →]  [Practice on Replit →]

User opens the link, codes the solution there, comes back and clicks:
[I completed this ✓]

No auto-grading — trust-based system. The goal is to make them code, not police them.
```

## Gemini API Integration

### Why Gemini (not Claude)?
- Gemini 1.5 Flash has a **generous free tier**: 15 RPM, 1 million tokens/day
- Perfect for quiz generation — occasional API calls, not real-time
- In BYOK mode, user brings their own Gemini key
- In subscription mode, your Gemini key is used (costs are very low)

### Gemini API Setup
```typescript
// lib/gemini.ts

const GEMINI_API_URL = 'https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent'

export async function generateVideoQuiz(
  transcript: string,
  videoTitle: string,
  apiKey: string
): Promise<QuizQuestion[]> {

  const prompt = `
You are an expert educator. Based on the following video transcript, 
generate a quiz to test understanding.

Video Title: "${videoTitle}"

Transcript:
${transcript.slice(0, 30000)}

Generate exactly 4 questions. Return ONLY a valid JSON array, no markdown, no explanation.
Format:
[
  {
    "type": "mcq",
    "question": "Question text here?",
    "options": ["Option A", "Option B", "Option C", "Option D"],
    "correct": 0,
    "explanation": "Why this answer is correct"
  },
  {
    "type": "truefalse", 
    "question": "Statement to evaluate",
    "options": ["True", "False"],
    "correct": 0,
    "explanation": "Explanation"
  },
  {
    "type": "fillblank",
    "question": "Complete this: The main purpose of ___ is to...",
    "options": ["word1", "word2", "word3", "word4"],
    "correct": 2,
    "explanation": "Explanation"
  },
  {
    "type": "mcq",
    "question": "Another question?",
    "options": ["A", "B", "C", "D"],
    "correct": 1,
    "explanation": "Explanation"
  }
]

Rules:
- Questions must be directly based on the transcript content
- MCQ options must all be plausible (no obviously wrong answers)
- Explanations must be helpful and educational
- Do not ask about timestamps, video production, or the presenter
- Focus on the actual concepts taught
`

  const res = await fetch(`${GEMINI_API_URL}?key=${apiKey}`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      contents: [{ parts: [{ text: prompt }] }],
      generationConfig: {
        temperature: 0.4,
        maxOutputTokens: 2048,
      }
    })
  })

  if (!res.ok) {
    const err = await res.json()
    if (res.status === 429) throw new Error('QUOTA_EXCEEDED')
    if (res.status === 400) throw new Error('INVALID_KEY')
    throw new Error('GEMINI_ERROR')
  }

  const data = await res.json()
  const text = data.candidates?.[0]?.content?.parts?.[0]?.text

  // Strip markdown code fences if present
  const clean = text.replace(/```json|```/g, '').trim()
  return JSON.parse(clean) as QuizQuestion[]
}

export async function generateFinalQuiz(
  courseTitle: string,
  videoTitles: string[],
  combinedTranscript: string,
  apiKey: string
): Promise<FinalQuizQuestion[]> {

  const prompt = `
You are an expert educator creating a comprehensive final assessment.

Course: "${courseTitle}"
Topics covered: ${videoTitles.join(', ')}

Combined transcript excerpts:
${combinedTranscript.slice(0, 50000)}

Generate a final course quiz with exactly 10 questions. 
Return ONLY valid JSON array, no markdown, no preamble.

Include these question types in this order:
- 3 MCQ questions (conceptual, harder than per-video quizzes)
- 2 short_answer questions (user types their understanding)
- 5 coding_practice questions (real coding challenges related to the course topic)

Format:
[
  {
    "type": "mcq",
    "question": "Advanced concept question?",
    "options": ["A", "B", "C", "D"],
    "correct": 0,
    "explanation": "Detailed explanation"
  },
  {
    "type": "short_answer",
    "question": "Explain in your own words: why is X important?",
    "sampleAnswer": "A good answer would mention: point1, point2, point3",
    "evaluationCriteria": ["mentions concept A", "explains why B", "gives example"]
  },
  {
    "type": "coding_practice",
    "question": "Write a function that does X",
    "context": "Use what you learned about Y in this course",
    "difficulty": "beginner",
    "platform_links": [
      { "name": "LeetCode", "url": "https://leetcode.com/problems/relevant-problem/" },
      { "name": "Replit", "url": "https://replit.com/new/nodejs" },
      { "name": "CodePen", "url": "https://codepen.io/pen/" }
    ],
    "hint": "Think about the Z concept from Video 3"
  }
]

For coding_practice questions:
- Make them directly relevant to the course topic
- Match difficulty to beginner/intermediate based on course content  
- LeetCode URL should link to a real, relevant problem if possible
- Replit/CodePen links can be generic new-file links
- Hints should reference actual course concepts

For short_answer questions:
- sampleAnswer should guide the AI evaluator, not be shown to user
- evaluationCriteria: 3 key points a good answer should cover
`

  const res = await fetch(`${GEMINI_API_URL}?key=${apiKey}`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      contents: [{ parts: [{ text: prompt }] }],
      generationConfig: {
        temperature: 0.3,
        maxOutputTokens: 4096,
      }
    })
  })

  if (!res.ok) throw new Error('GEMINI_ERROR')
  const data = await res.json()
  const text = data.candidates?.[0]?.content?.parts?.[0]?.text
  const clean = text.replace(/```json|```/g, '').trim()
  return JSON.parse(clean) as FinalQuizQuestion[]
}

export async function evaluateShortAnswer(
  question: string,
  userAnswer: string,
  sampleAnswer: string,
  criteria: string[],
  apiKey: string
): Promise<{ score: number, feedback: string, criteriamet: string[] }> {

  const prompt = `
Evaluate this student answer for a learning quiz.

Question: "${question}"
Student's answer: "${userAnswer}"
Reference answer: "${sampleAnswer}"
Evaluation criteria: ${criteria.join(', ')}

Score the answer from 0-100 and provide constructive feedback.
Return ONLY valid JSON, no markdown:
{
  "score": 75,
  "feedback": "Good answer! You correctly mentioned X and Y. You could improve by also explaining Z.",
  "criteriaMet": ["mentions concept A", "gives example"]
}

Be encouraging but honest. If answer is blank or irrelevant, score 0.
`

  const res = await fetch(`${GEMINI_API_URL}?key=${apiKey}`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      contents: [{ parts: [{ text: prompt }] }],
      generationConfig: { temperature: 0.2, maxOutputTokens: 512 }
    })
  })

  if (!res.ok) throw new Error('GEMINI_ERROR')
  const data = await res.json()
  const text = data.candidates[0].content.parts[0].text
  const clean = text.replace(/```json|```/g, '').trim()
  return JSON.parse(clean)
}
```

## Database Schema (Add to existing schema.ts)

-- Quiz attempts table
quiz_attempts
  id                uuid        PRIMARY KEY DEFAULT gen_random_uuid()
  user_id           uuid        REFERENCES users(id) ON DELETE CASCADE

## Scoring System

```
Per-video quiz scoring:
  Each question: correct = 1 point, wrong = 0
  Final score = (correct / total) × 100
  Passing score: 60% (not enforced — just shown)

Final quiz scoring:
  MCQ questions:        20 points each (3 × 20 = 60 pts)
  Short answer:         10 points each (2 × 10 = 20 pts, Gemini evaluates)
  Coding practice:      4 points each  (5 × 4  = 20 pts, trust-based self-report)
  Total:                100 points

Score display:
  0–49:   "Keep practicing — revisit the videos 💪"
  50–69:  "Good effort! Review the weak areas"
  70–84:  "Well done! Solid understanding 🎯"
  85–100: "Excellent! You've mastered this course 🏆"

Certificate unlock:
  → Always unlocked on course completion (watching all videos)
  → Final quiz score shown ON the certificate
  → "Score: 85/100" shown in certificate stats row
  → If quiz skipped → certificate shows "Quiz: Not taken"
```

## Notes for AI Assistants

1. **Gemini API is REST-based** — no npm SDK needed, use native fetch()
2. **Always strip markdown fences** from Gemini response before JSON.parse()
3. **Retry once** if JSON.parse() fails — Gemini occasionally returns malformed JSON
4. **Transcript truncation** — send max 30,000 chars for video quiz, 50,000 for final quiz
5. **Coding questions are trust-based** — no auto-grading, just a checkbox
6. **Quiz is always optional** — never block course completion or certificate behind quiz
7. **Store full questions in DB** — do not regenerate on page refresh, use cached attempt
8. **One attempt stored per video** — if user retakes, overwrite previous attempt
9. **BYOK key storage** — store in users table as gemini_api_key column (add in migration)
10. **Short answer min length** — enforce 30 word minimum before allowing submission
